# 纳米级计划：GitHub Actions CI 集成

## 目标
自动验证后续代码提交的正确性，通过 GitHub Actions 执行完整的后端测试、前端单元测试和端到端测试。

## 涉及文件
- `[NEW] .github/workflows/ci.yml`

## 原子化任务拆解

### 1. 结构与环境配置
- **动作**：创建 `.github/workflows/ci.yml`
- **内容**：
  - 定义 `name: GoDeployer CI`
  - 定义触发条件 `on: [push, pull_request]` 对 `main` 分支生效

### 2. Job: Backend Tests (后端测试)
- **动作**：配置 `backend-test` Job
- **内容**：
  - 使用 `ubuntu-latest`
  - 安装 Go 1.20
  - 执行 `go mod download`
  - 执行 `go test -v -race ./...`

### 3. Job: Frontend Unit Tests (前端组件/单元测试)
- **动作**：配置 `frontend-unit` Job
- **内容**：
  - 使用 `ubuntu-latest`
  - 安装 Node.js 18
  - `cd web && npm ci`
  - 执行 `npm run test` (Vitest)

### 4. Job: E2E Tests (端到端综合测试)
- **动作**：配置 `e2e-test` Job
- **内容**：
  - 使用 `ubuntu-latest`
  - 依赖上述两个 Job (`needs: [backend-test, frontend-unit]`)
  - 安装 Go 1.20 与 Node.js 18
  - 安装 Playwright 浏览器依赖：`npx playwright install --with-deps`
  - `cd web && npm ci`
  - 启动测试：`npm run test:e2e` (注意此命令依赖上级目录存在可运行的 `main.go` 源码，我们已经在 `playwright.config.ts` 中配置了通过 `go run main.go` 后台启动)
  - 保留 Playwright Report 作为 Artifact 上传以便排错

## 验收标准
- 物理生成 `.github/workflows/ci.yml` 文件。
- 代码无任何业务侵入。
