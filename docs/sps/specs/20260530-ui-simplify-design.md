# GoDeployer UI 简化和稳定性重构设计

日期: 2026-05-30
状态: 已批准
关联: ADR 见 docs/sps/decisions/

## 1. 目标

将 GoDeployer 前端从当前不稳定、业务流程混乱的状态，重构为**稳定可靠、流程清晰、易用**的系统。

核心原则：
- **能用优先**：保证部署全流程可走通，不出错
- **简化优先**：每个组件一个职责，每个按钮一个动作
- **错误可见**：每次失败都给出明确原因和操作建议

## 2. 组件架构

### 2.1 拆分方案

将 1700+ 行 Dashboard.vue 拆分为 6 个子组件 + 1 个精简组合器：

| 组件 | 职责 | 行数预估 | 测试重点 |
|---|---|---|---|
| ProjectSidebar.vue | 项目列表 + 环境选择，纯展示，emit 事件 | ~80 | 选中高亮、事件通信 |
| DeployForm.vue | 版本选择（分支/Tag/Commit）+ 备注 + 部署/预览按钮 | ~150 | 表单校验、API 调用 |
| DeployHistoryTable.vue | 部署历史表格 + 回滚/对比/日志按钮 | ~100 | 数据渲染、操作按钮启用逻辑 |
| LogTerminal.vue | 日志弹窗 + WebSocket 管理 + 断线重连 | ~120 | WS 连接/断开/重试 |
| DiffDialog.vue | Diff 对比弹窗 + 只读文件树 | ~200 | 大文件截断、空数据 |
| UserSettingsDialog.vue | 账号 Git 白名单配置弹窗 | ~80 | 表单保存/加载 |
| Dashboard.vue（精简） | 组合容器，引入子组件，协调共享状态 | ~200 | 集成测试 |

### 2.2 布局调整

- 项目侧栏和环境选择整合到同一侧栏（当前两者分离，交互断裂）
- 部署表单固定在右上工作区，不再和 Tab 切换混淆
- 去除 Diff 弹窗中嵌套部署确认逻辑，Diff 只做纯展示
- 所有页面统一暗色主题

## 3. 数据流修复

### 3.1 消除 Mock 数据

当前问题：`fetchHistory` 的 catch 块返回虚假数据，掩盖后端接口故障。

修复：catch 块直接 `ElMessage.error` + 表格显示「暂无部署记录」空状态，不返回假数据。

### 3.2 单一状态机

当前：3 个 boolean flag 状态交织（isPreDeploying / pendingDeployEnv / activeTask）。

改为单一枚举：

```
deployPhase: 'idle' | 'previewing' | 'confirming' | 'deploying' | 'done' | 'error'
```

任何时刻只有一个状态，消除状态冲突。

### 3.3 去除自动触发 watch

当前：watch 监听 deployForm.branch 自动调用 preview_diff 接口，切换分支导致不必要的请求和 UI 闪烁。

修复：去掉 watch，改为用户点「预览 Diff」按钮才请求。切换分支只更新下拉框。

### 3.4 WebSocket 稳定性

修复：
- 每 30s 心跳 ping
- 断线后 3 次重试：间隔 1s / 3s / 5s
- 全部失败后 fallback 到 HTTP 轮询（每 2s 查询日志 + 检查任务状态）
- 组件卸载时清理所有连接和定时器

## 4. Diff 弹窗简化

### 4.1 当前问题

- 双模式对比（Live Diff / Git Log）用户需要理解何时哪种可用
- 文件树 checkbox 用于部署前文件过滤，操作复杂
- 部署确认按钮嵌入 Diff 弹窗，职责混淆

### 4.2 简化

| 当前 | 改为 |
|---|---|
| 双模式切换 | 单一模式：Git Log Diff（始终可用）。Live Diff 入口移至历史表格行操作 |
| 文件树 checkbox | 只读文件树（仅查看）。文件排除由项目配置的 exclude 控制 |
| Diff 弹窗内确认部署 | Diff 仅做展示。部署始终走主表单的「部署上线」按钮 |
| diff2html 渲染 | 保留截断保护（100KB），但改为纯文本 diff，确保绝不卡死 |

## 5. 错误处理

### 5.1 全局错误拦截

在 axios 拦截器中统一处理：
- 401 → 跳转到登录页
- 403 → 提示权限不足
- 500 → 统一弹窗，附带错误详情

### 5.2 错误信息规范

```
格式：「操作名称：具体原因，建议操作」

示例：
「部署失败：SSH 连接 192.168.1.100:22 超时，请检查目标服务器是否在线」
「获取对比失败：Git 仓库状态异常，请在服务器上执行 git status」
```

## 6. 主题统一

- `UserManagement.vue` 改为暗色主题，与 Dashboard 一致
- 提取公共 Header 组件，登录后所有页面复用
- 引入统一的 CSS 变量文件，各页面引用

## 7. 测试策略

| 层面 | 内容 |
|---|---|
| 单元测试 | 每个子组件独立 mount，测试 props/events/表单校验 |
| 集成测试 | 模拟完整部署流程：选项目 → 选版本 → 部署 → 看日志 |
| E2E | 补全核心流程用例，不改变现有 Playwright 框架 |

## 8. 实施顺序

1. **Phase 1：修复核心数据流**（消除 Mock、单一状态机、去除 watch 自动触发）
2. **Phase 2：组件拆分**（提取 6 个子组件，测试通过）
3. **Phase 3：Diff 弹窗简化** + 主题统一
4. **Phase 4：错误处理规范化** + WebSocket 稳定性
5. **Phase 5：测试补全**
