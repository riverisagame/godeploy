# 架构决策记录 (ADR) - 文档结构设计

## 背景
用户要求对 `deployer` 项目进行功能分析，并生成开发文档、手册和产品说明书。

## 决策
我们将文档分为三个维度，分别存储在 `d:\claudeprj\deploy\doc` 目录下。

### 1. 开发文档 (DEVELOPMENT.md)
- **受众**: 开发者、贡献者。
- **内容**: 
  - 系统架构设计。
  - 核心组件解析 (`Deployer`, `Host`, `Task`, `Collection`)。
  - 插件/配方 (Recipe) 系统机制。
  - 测试套件说明。

### 2. 使用手册 (MANUAL.md)
- **受众**: DevOps 工程师、系统管理员。
- **内容**:
  - 快速开始。
  - 部署流程 (`deploy.php`) 配置。
  - 任务 (Task) 定义与执行。
  - 并发执行与主机管理。
  - 常见问题排查。

### 3. 产品说明书 (PRODUCT_SPEC.md)
- **受众**: 技术决策者、新用户。
- **内容**:
  - 产品定位与核心价值。
  - 功能特性矩阵。
  - 相比于其他工具（Ansible, Capistrano）的优势。
  - 典型应用场景。

## 验证方式
- 确保文档内容与源码逻辑一致。
- 参考 Deployer 8.x 官方文档确保术语准确。
