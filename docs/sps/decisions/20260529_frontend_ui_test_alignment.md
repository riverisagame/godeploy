# 需求名称：前端 UI 测试 100% 按钮点击正确性覆盖
**日期**: 2026-05-29

## 1. 需求分析
**用户目标**：保证前端网页（Dashboard、Login、UserManagement 等）的**每一个**按钮不仅视觉上可见，还能真实被点击且不抛出未处理的异常，流程功能正确流转。
**当前现状**：之前我们通过 `test_deploy_e2e.js` 在 API 层模拟了全流程调用流转。但该脚本是在纯 Node 环境下直接发请求，**并没有点击网页 DOM 中的按钮**。这确实是一个遗漏。

## 2. 功能影响面评估
- **影响范围**：由于只增加前端的 `vitest` / `@vue/test-utils` 组件测试，它会在构建和测试阶段发挥作用，对实际已部署的业务代码不会产生任何侵入或破坏。
- **并发与响应影响**：无影响，因为这是离线静态验证。

## 3. 测试用例补齐目标 (对齐)
- **Login 模块**：点击“登录”按钮触发验证，模拟正确的账号密码导致 router push；模拟错误的则展示 Error message。
- **UserManagement 模块**：点击“新建用户”弹出弹窗；点击“解除绑定”正常调用接口。
- **Dashboard 模块**：点击“环境标签”能正常切换选中的环境；点击“部署”按钮能弹出部署提示并正确调用 `/api/tasks`。
