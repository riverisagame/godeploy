# GoDeploy 手把手从零搭建指南

> 本文档假定你是一名刚接触编程的初二学生。每一步都会详细解释"为什么这样做"和"这样做是在干什么"。
> 即使你完全不懂 Go、Vue、WSL、Linux 这些术语，按照步骤操作也能成功。

---

## 零、这到底是什么？

**GoDeploy** 是一个"代码自动发布系统"。

想象一个场景：你和小伙伴一起写了一个网站，代码存在 GitHub / Gitee 上。
每次修改完代码，你需要：
1. 登录服务器
2. 拉取最新代码
3. 运行编译命令
4. 重启服务

这几个步骤很繁琐、容易出错。GoDeploy 的作用就是：**在网页上点一个按钮，自动帮你完成以上所有步骤**。

它还能：
- 显示每次发布的记录（谁、什么时候、发布了什么）
- 对比新旧代码的差异（Diff 对比）
- 一键回滚到上一个版本（后悔药）
- 多个人用不同权限操作（管理员可以管所有人，运维只能部署某些项目，测试只能看）

## 一、你需要准备什么

### 硬件要求

一台 Windows 10/11 电脑，内存 8GB 以上。

### 软件清单

我们需要安装以下软件。如果你是第一次接触编程，请务必按顺序安装。

| 软件 | 作用 | 下载地址 |
|------|------|---------|
| WSL2 Debian | 在 Windows 里跑一个 Linux 系统 | Windows 自带，一条命令安装 |
| Go 1.25+ | 编译后端程序的语言 | https://go.dev/dl/ |
| Node.js 20+ | 编译前端网页的工具 | https://nodejs.org/ |
| Git | 管理代码版本 | https://git-scm.com/ |

## 二、第一步：安装 WSL (Windows 子系统 Linux)

### 为什么需要 WSL？

后端程序是在 Linux 上运行的，但你的电脑是 Windows。
WSL 让你在 Windows 里直接运行一个真正的 Linux 系统，不需要虚拟机，不需要双系统。

### 安装步骤

**① 用管理员身份打开 PowerShell**

按键盘上的 `Win + X` 键，选择"终端管理员"或"Windows PowerShell (管理员)"。

**② 输入以下命令，然后按回车：**

```powershell
wsl --install -d Debian
```

这条命令的意思是：安装 WSL 并下载 Debian 这个 Linux 发行版。
Debian 是一个稳定、轻量的 Linux 系统，适合开发和服务器使用。

**③ 等待安装完成**

下载大约 400MB，取决于网速需要 5-15 分钟。

**④ 安装完成后，会弹出一个黑色窗口，要求你设置用户名和密码：**

```
Enter new UNIX username: 你的英文名
New password: 你的密码（输入时不显示，这是正常的）
Retype new password: 再输一遍
```

> **重要**：密码输入时屏幕不会显示任何字符，这是 Linux 的安全设计。输完按回车即可。

**⑤ 安装必要工具**

设置完密码后，会进入命令行界面。依次输入以下命令：

```bash
# 更新软件包列表
sudo apt-get update

# 安装部署需要的工具
sudo apt-get install -y git sqlite3 jq curl rsync lsof openssh-server

# 启动 SSH 服务（部署时需要）
sudo service ssh start
```

每行命令解释：
- `sudo apt-get update`：更新 Debian 的软件列表，"sudo" 是"以管理员身份执行"的意思
- `sudo apt-get install -y ...`：安装后面列出的所有软件，`-y` 表示自动确认
- `sudo service ssh start`：启动 SSH 服务，让程序可以通过 SSH 连接到"目标服务器"

**⑥ 关闭 WSL 窗口**

直接点 X 关闭即可。WSL 会自动保存状态。

## 三、第二步：安装 Go 语言

### 为什么需要 Go？

后端程序是用 Go 写的，需要通过 Go 编译器把源代码变成可以运行的程序。

### 安装步骤

**① 下载 Go**

打开浏览器，访问 `https://go.dev/dl/`

找到 `go1.25.0.windows-amd64.msi`（Windows 64位安装包），点击下载。

**② 安装**

双击下载的文件，一直点"Next"直到完成。所有选项保持默认即可。

**③ 验证安装**

打开 PowerShell（Win + X → 终端），输入：

```powershell
go version
```

如果显示类似 `go version go1.25.0 windows/amd64`，说明安装成功。

## 四、第三步：安装 Node.js

### 为什么需要 Node.js？

前端网页是用 Vue 框架写的，需要通过 Node.js 的工具链把 `.vue` 文件编译成浏览器能看懂的 HTML/JS/CSS。

### 安装步骤

**① 下载 Node.js**

打开浏览器，访问 `https://nodejs.org/`

下载 **LTS 版本**（长期支持版，更稳定）。推荐 20.x 或 22.x。

**② 安装**

双击下载的文件，一直点"Next"直到完成。

**③ 验证安装**

打开 PowerShell，输入：

```powershell
node --version
npm --version
```

应该分别显示 `v20.x.x` 和 `10.x.x` 之类的版本号。

## 七、第六步：编译项目

### 两种方式

| 方式 | 速度 | 适用场景 |
|------|------|---------|
| 快速启动脚本 | 3秒 | 快速演示、开发测试 |
| 手动分步编译 | 可控 | 了解每个步骤 |

### 方式一：一键启动（推荐）

打开 PowerShell，输入：

```powershell
wsl -d Debian bash -c "cd /mnt/d/claudeprj/deploy && bash scripts/demo.sh all"
```

这条命令做了以下事情：
1. 检查依赖是否安装
2. 生成 3 个模拟的 Git 仓库（think/webman/CRMEB）
3. 编译前端（通过 Windows 原生 npm，约 2 秒）
4. 编译后端（通过 Windows 原生 Go 交叉编译，约 1 秒）
5. 启动服务
6. 通过 API 模拟用户操作，生成演示数据
7. 显示访问地址和账号

如果 Windows 工具不可用，会自动回退到 WSL 内编译（慢 10 倍）。

### 方式二：手动分步编译

如果你想理解每一步在做什么：

**① 编译前端**

```powershell
# 进入前端目录
cd D:\claudeprj\deploy\web

# 安装依赖（第一次需要，之后不用）
npm install

# 编译
npm run build
```

编译完成后，`web/dist/` 目录下会生成 `index.html` 和一堆 JS/CSS 文件。

**② 复制前端产物**

```powershell
cd D:\claudeprj\deploy
Remove-Item -Recurse godeployer\dist -Force
Copy-Item -Recurse web\dist godeployer\dist
```

为什么需要复制？因为 Go 代码中有这一行：

```go
//go:embed dist
var embeddedFiles embed.FS
```

这行代码的意思是：编译 Go 程序时，把 `godeployer/dist/` 目录下所有文件嵌入到最终的 `.exe` 文件中。这样只需要一个文件就能同时提供前端网页和后端服务。

**③ 编译后端**

```powershell
# 设置环境变量（只对当前命令生效）
#   GOOS=linux    → 编译目标系统是 Linux
#   GOARCH=amd64  → 编译目标是 64 位
#   CGO_ENABLED=0 → 禁用 CGO（纯 Go 编译，跨平台无依赖）
$env:GOOS="linux"
$env:GOARCH="amd64"
$env:CGO_ENABLED="0"

# 编译
go build -o godeployer_demo_bin .
```

这个命令生成文件 `godeployer_demo_bin`，是一个 Linux 可执行文件。

## 八、第七步：配置文件说明

### 主配置文件：demo_config.yaml

```yaml
global:
  server_port: 8080              # 服务端口号
  sqlite_path: demo_deployer.db  # 数据库文件位置
  project_config_dir: demo_projects.d  # 项目配置目录
  jwt_secret: demo-secret-key    # JWT 密钥（生产环境必须改）
```

### 项目配置：demo_projects.d/thinkphp.yaml

```yaml
id: thinkphp-web                 # 项目唯一标识
name: ThinkPHP 后端框架          # 显示名称
repo: ./demo_workspace/gitee_demo/think  # Git 仓库路径（相对/绝对）
exclude:                         # 同步时忽略的文件
  - .git
  - vendor
  - runtime
environments:                    # 部署环境列表
  - id: staging                  # 环境标识
    name: 测试环境               # 显示名称
    default_branch: develop      # 默认分支
    keep_releases: 3             # 保留最近几个发布版本
    servers:                     # 目标服务器列表
      - host: localhost          # 服务器地址
        port: 2222               # SSH 端口
        user: dan                # SSH 用户名
        deploy_to: /tmp/demo_deploy/thinkphp/staging  # 部署路径
  - id: production
    name: 生产环境
    default_branch: master
    keep_releases: 5
    servers:
      - host: localhost
        port: 2222
        user: dan
        deploy_to: /tmp/demo_deploy/thinkphp/production
```

每个字段的含义：
- `id`: 代码中引用的唯一名称，用字母、数字和连字符
- `name`: 网页上显示的中文名
- `repo`: Git 仓库路径，支持本地路径（`./xxx`）和远程 URL
- `exclude`: 同步时跳过的文件，通常包括 `.git`、`vendor` 等
- `environments`: 可以有多个环境（测试/预发/生产），每个环境可以有不同的服务器和配置
- `servers`: 目标服务器列表，支持多台服务器同时部署
- `keep_releases`: 只保留最近 N 个版本，旧的自动删除（节省磁盘）

## 九、功能测试指南

打开浏览器，访问 `http://localhost:8080`：

### 1. 登录

在登录页输入 `admin` / `admin123`，点击"登录"。

### 2. 查看项目

登录后，左侧显示 4 个项目：
- CRMEB 商城系统
- Test Small Proj
- ThinkPHP 后端框架
- Webman 微服务接口

点击 ThinkPHP 后端框架。

### 3. 查看发布表单

页面中部左侧是"触发部署"区域，包含：
- **发布目标类型**：分支 / Tag / 历史 Commit（三个按钮选一个）
- **选择分支/Tag**：下拉框，显示所有分支和 Tag
- **发布备注/说明**：必填，红色星号标记
- **触发上线** 按钮：点击开始部署
- **预览 Diff** 按钮：先看代码变化再决定是否部署

### 4. 触发一次部署

1. 确保"分支"被选中
2. 在"选择分支/Tag"下拉框中选择 `develop`
3. 在"发布备注/说明"输入框中填入 `测试部署：第一次上线`
4. 点击"触发上线"
5. 系统弹出 Diff 对比窗口，可以预览将要部署的代码变化
6. 注意：在这个窗口点击"确认并部署"才会真正执行

### 5. 查看发布历史

页面右侧是"部署与审计历史"表格，可以看到：
- 每次部署的 ID、Release 版本号
- 部署了哪个 Commit
- 是谁操作的
- 状态（部署成功/失败/进行中/已回滚）
- 时间

每一行有三个操作按钮：
- **回滚**：回到上一个版本（只有"部署成功"的任务可以回滚）
- **对比**：查看这次部署的代码 Diff
- **日志**：查看部署过程日志（5 步详细输出）

### 6. 查看 Diff 对比

点击任意一行的"对比"按钮，弹出全屏代码对比窗口：

- **左侧**：变更文件列表（树形结构）
- **右侧**：代码差异视图
- **顶部按钮**：
  - 与线上对比 (Live Diff) / 本地变更 (Git Log Diff)：切换对比模式
  - 竖向对比 / 横向对比：切换视图布局
- 新增的代码行以深绿色背景显示
- 删除的代码行以深红色背景显示

### 7. 查看部署日志

点击任意一行的"日志"按钮，弹出终端风格窗口，显示部署 5 步的详细日志：

```
[2026-05-30 21:03:26] Updating local bare repo cache...
[2026-05-30 21:03:27] Step 1: Cloning repository locally from cache...
[2026-05-30 21:03:28] Step 2: Checking out target commit/branch...
[2026-05-30 21:03:29] Step 3: Executing local build hooks...
[2026-05-30 21:03:30] Step 4 [Phase1]: Synchronizing files to remote server...
[2026-05-30 21:03:31] Step 5 [Phase2]: Switching active symlink...
[2026-05-30 21:03:31] Deployment completed successfully!
```

### 8. 测试回滚

1. 找到一条"部署成功"的记录
2. 点击"回滚"按钮
3. 弹出确认对话框
4. 点击"确认回滚"
5. 该任务状态变为"已回滚"

### 9. 测试必填验证

1. 清空"发布备注/说明"输入框
2. 点击"触发上线"
3. 应该弹出一条黄色警告：**"请输入发布备注/说明"**
4. 部署不会执行

### 10. 切换用户测试权限

1. 点击右上角"退出登录"
2. 用 `deployer` / `deploy123` 登录
3. 左侧应该只显示 ThinkPHP 和 Webman（deployer 没有 CRMEB 的权限）
4. 用户管理按钮也不显示（只有 admin 能看到）

## 十、后台管理功能

### 用户管理（仅 admin）

点击顶部"用户管理"按钮，进入用户管理页面。

可以：
- 查看所有用户列表
- 创建新用户（用户名、密码、角色、允许访问的项目）
- 编辑/删除用户

一个用户的完整配置：
- **角色**：admin（全部权限）、deployer（可部署指定项目）、viewer（只能看）
- **允许项目**：逗号分隔的项目 ID 列表，如 `thinkphp-web,webman-api`
- **限制 Git 作者**：开启后只能部署指定作者提交的代码

### 系统自愈清理（仅 admin）

点击顶部"系统自愈清理"按钮，自动：
- 清理过期的部署记录
- 删除孤儿垃圾文件
- 释放磁盘空间

### 账号配置

点击顶部"账号配置"按钮，可以：
- 修改当前用户的 Git 提交人绑定
- 设置 Git 作者白名单限制

## 十一、常见错误及解决方法

### 错误 1：`wsl: command not found`

**原因**：WSL 没有安装。

**解决**：以管理员身份打开 PowerShell，运行：
```powershell
wsl --install -d Debian
```

### 错误 2：`go: command not found`

**原因**：Go 没有安装或没有加入 PATH。

**解决**：重新安装 Go，安装完成后重启 PowerShell。

### 错误 3：`npm: command not found`

**原因**：Node.js 没有安装。

**解决**：去 https://nodejs.org 下载安装。

### 错误 4：`ERR_CONNECTION_REFUSED`

**原因**：后端服务没有启动。

**解决**：
```bash
wsl -d Debian bash -c "cd /mnt/d/claudeprj/deploy && bash scripts/demo.sh start"
```

### 错误 5：`cannot execute binary file: Exec format error`

**原因**：试图在 WSL 中运行 Windows 编译的 `.exe` 文件，格式不兼容。

**解决**：确保编译时设置了 `GOOS=linux GOARCH=amd64`。

### 错误 6：`permission denied`（SSH 连接失败）

**原因**：WSL 中的 SSH 服务没有启动。

**解决**：
```bash
wsl -d Debian bash -c "sudo service ssh start"
```

### 错误 7：`repository does not exist`（部署失败）

**原因**：bare repo 缓存还没有创建。

**解决**：这是正常的，首次部署会自动创建缓存。后续部署就快了。

### 错误 8：`8080 端口被占用`

**原因**：上一个 godeployer 进程还在运行。

**解决**：
```bash
wsl -d Debian bash -c "killall godeployer_demo_bin; bash scripts/demo.sh start"
```

## 十二、目录清理

如果想把一切还原到初始状态：

```bash
wsl -d Debian bash -c "cd /mnt/d/claudeprj/deploy && bash scripts/demo.sh stop && rm -rf demo_workspace demo_deployer.db demo_logs godeployer_demo.log"
```
